# Homology certificate TODO

Updated: 2026-05-12, end of night.

## Current state

The SNF certificate path is working as a prototype.

- `snf A` adds `cert : CertificateSNF A` to the local context.
- `snf A as h` adds the certificate under a chosen name.
- `mkSNFCertExpr` is the reusable backend: it calls Sage, decodes matrices, builds the `CertificateSNF` expression, and Lean checks the certificate fields.
- SNF examples now cover:
  - full-rank matrix;
  - rank-deficient matrix;
  - boundary matrices `boundaryK triangleFFC 1` and `boundaryK triangleFFC 2`;
  - a rectangular boundary matrix.
- `boundaryK` has been cleaned up:
  - no separate `boundaryKInt`;
  - `boundaryRowCount` is a reducible helper;
  - `boundaryK X k` now has the homological indexing convention, i.e. `∂ₖ : Cₖ → Cₖ₋₁`.
- `Cultivar.lean` no longer imports the deleted examples aggregator.

The homology certificate data model is in `Cultivar/Homology/Basic.lean`:

- `HomologyQuotientCert dk dk1`
  - `certK : CertificateSNF dk`
  - `hCC : dk * dk1 = 0`
  - `M : Matrix (Fin (n - certK.r)) (Fin p) R`
  - `hM : M = cyclePresentationMatrix certK dk1`
  - `certM : CertificateSNF M`
- `CertificateHomology X k` wraps the quotient certificate for
  `boundaryK X k` and `boundaryK X (k + 1)`.

## Important design conclusion from tonight

Do not build the final homology workflow by writing local proof terms like:

```lean
have certK := ...
let M := cyclePresentationMatrix certK dk1
snf M
```

That is a bad interface for the current SNF tactic. The user-facing `snf` tactic
serializes a matrix by compiled evaluation of `matStringList A`, and compiled
evaluation needs a closed expression. A local `M` depending on local `certK`
contains proof-local free variables, which causes errors like:

```text
(kernel) declaration has free variables '_tmp✝'
```

The right direction is:

1. The homology tactic elaborates `X` and `k`.
2. It builds closed expressions for `dk := boundaryK X k` and
   `dk1 := boundaryK X (k + 1)`.
3. It calls `mkSNFCertExpr dk` to get `certK`.
4. It constructs the presentation matrix expression itself:
   `cyclePresentationMatrix certK dk1`.
5. It obtains or constructs a closed/usable expression for that matrix and calls
   the SNF backend on it, not the user-facing `snf` tactic on a proof-local let.
6. It assembles `HomologyQuotientCert` / `CertificateHomology`.

If step 5 remains painful because `M` depends on the generated certificate term,
use the Sage homology witness path: have Sage return `M`, verify
`M = cyclePresentationMatrix certK dk1` in Lean by `native_decide`, and certify
Sage's returned `M` with `mkSNFCertExpr`.
